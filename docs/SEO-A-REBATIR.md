# SEO agressif - Hygiene informatique positive

Date: 2026-07-02

L'ancienne strategie marketing/SEO a ete archivee dans `docs/archive/marketing-2026-07-02/`.

La nouvelle base active vise une strategie organique Google + AI sur 90 jours autour d'un hub central:

- `/hygiene-informatique`
- `/loi-25`
- `/cybersecurite`
- `/intelligence-artificielle`

## Positionnement

Une bonne hygiene informatique n'est pas seulement defensive. Pour une PME ou un OBNL, elle sert a:

- mieux connaitre les donnees collectees;
- inspirer confiance aux clients, parents, employes et partenaires;
- reduire les erreurs et les acces inutiles;
- rendre les sauvegardes, incidents et mises a jour plus previsibles;
- adopter l'IA avec des regles simples et une validation humaine.

## Architecture active

- Hub: `/hygiene-informatique`
- Guides transversaux: gestes d'hygiene, score PME, confiance client
- Cluster Loi 25: resume, inventaire des donnees, formulaires, conservation/suppression, acces aux renseignements
- Cluster cybersecurite: MFA, sauvegardes, acces employes, courriels frauduleux, mises a jour, procedure incident
- Cluster IA: charte IA, donnees sensibles, validation humaine, choix d'outils, cas d'usage productifs

## Regles d'execution

- Les diagnostics gratuits restent le CTA principal.
- Le contact humain reste le CTA secondaire.
- Les pages doivent etre utiles, visibles dans le sitemap et coherentes avec leur JSON-LD.
- Pas de contenu mince publie seulement pour remplir un calendrier.
- Pas de promesse de conformite garantie, de cybersecurite garantie ou d'automatisation IA magique.

## Suivi organique sobre

- Suivre Search Console: impressions, CTR, requetes, pages.
- Suivre les conversions organiques via les wizards et le contact.
- Prioriser les iterations sur les titles/metas et les introductions si une page obtient des impressions sans clics.
